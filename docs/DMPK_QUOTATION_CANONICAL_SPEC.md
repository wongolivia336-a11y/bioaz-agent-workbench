# DMPK 报价 Agent Canonical Spec

> 版本：2026-07-13  
> 依赖通用规范：`G:\实习\bioaz-design-system\docs\BIOAZ_AGENT_WORKBENCH_CANONICAL_SPEC.md`  
> 目的：把 DMPK 报价 Agent 已确认的页面结构、交互、字段收集、右侧 panel、导出预览和风格细节固化，后续不再反复沟通基础形态。

## 1. 产品定位

DMPK 报价 Agent 是一个看起来智能的结构化报价计算器。

它不是：

- 通用 Agent。
- Coding Agent。
- Canvas 编辑器。
- 传统后台表单。

它是：

- 对话驱动。
- 参数结构化。
- 规则校验。
- 报价单生成。
- 产物预览与导出。

核心任务：

```text
用户输入自然语言需求
→ Agent 识别业务线和已知字段
→ 底部补全卡一次性收集缺失参数
→ 用户发送结构化参数
→ Agent 校验字段完整性和计价规则
→ 报价前确认
→ 生成 Word 报价单与 Excel 报价明细
```

## 2. 页面结构

DMPK 必须沿用 BioAZ Agent Workbench 三栏结构。

```text
Left Sidebar
Center Conversation
Right Panel Stack
```

### 左侧 Sidebar

直接复用肿瘤报告 Agent sidebar。

必须包含：

- BioAZ logo。
- 折叠按钮。
- 新建对话按钮。
- 搜索按钮。
- 项目组。
- 项目组展开/折叠。
- 项目 hover 后出现省略号和编辑入口。
- 项目操作菜单。
- chat row。
- 状态轻提示。
- 底部账号区。

DMPK 项目组示例：

```text
DMPK 报价
  Balb/c nude 报价...
```

不要重新设计 sidebar。

### 中间主工作区

中间包含：

- Topbar / breadcrumb。
- Agent header。
- Chat scroller。
- Composer dock。

Chat scroller 中只放：

- Agent 回复。
- 用户气泡。
- 折叠 activity chain。
- 生成后的核心产物入口。

不要把完整参数卡、补全卡、报价前确认卡放进聊天气泡里。

### 右侧 Panel Stack

右侧固定列包含：

```text
参数收集 panel（常驻）
secondary panel（hover/pin 出现）
```

右侧不是 tab 切换。

## 3. 用户流程

### 3.1 新对话进入

Agent 首句：

```text
你好，我是 DMPK 报价智能体。你可以直接描述检测类型、动物种属、数量、周期和采血点，我会先识别已知参数，再用一张补全卡一次性收集缺失项。
```

用户可输入自然语言，例如：

```text
PK小分子，SD大鼠，每组2只，2组，试验周期1周，试验周期内3个非加班时间点。
```

### 3.2 Agent 识别

Agent 应识别：

- DMPK / PK 业务线。
- 分子类型。
- 动物种属。
- 每组动物数。
- 组数。
- 试验周期。
- 采血点。

Agent 回复示例：

```text
已识别 DMPK / PK 检测、小分子、SD 大鼠、每组 2 只、2 组、试验周期 1 周和 3 个非加班采血点。还需要补充生物分析和报告与报价参数。
```

### 3.3 参数补全

缺失参数通过 composer 上方的参数补全卡一次性收集。

不要：

- 一个问题一个问题追问。
- 按类别收集完就自动发送。
- 每页填完自动切下一页。
- 把补全卡放到聊天气泡里。

要：

- 一张补全卡。
- 四个固定业务分页。
- 用户手动切页。
- 选项写入 composer chip。
- 全部缺失项都写入 chip 后再统一发送。

### 3.4 报价前确认

当必填参数齐全后：

- Agent 在对话里提示参数已齐全。
- Composer 上方出现“报价前确认”任务卡。
- 用户可点“预览全部参数”打开 modal。
- 用户可点“生成报价单”开始生成。

报价前确认卡必须在 composer 上方，不进入 chat 气泡。

### 3.5 生成报价

生成后：

- 主对话显示生成结论。
- 右侧参数 panel 仍常驻。
- 产物列表通过 secondary panel hover/pin 出现。
- Word 与 Excel 可分别预览。

## 4. 参数补全卡

### 4.1 位置

参数补全卡位于 composer shell 上方，属于底部输入工作区。

```text
参数补全卡
Composer Shell
  ├─ 参数 chips
  ├─ 输入框
  └─ 发送按钮
```

### 4.2 分页

分页是按业务类型固定分的，不按条数自动切片。

固定四页：

1. 检测类型。
2. 动物实验。
3. 生物分析。
4. 报告与报价。

规则：

- 默认可从当前最相关业务页开始，但不要自动跳页。
- 用户自己点页签、上一页、下一页。
- 每页展示该业务页下仍缺失的字段。
- 某页字段已齐全时，展示轻提示：`该页参数已齐全，可切换下一页继续补全。`

### 4.3 Header

Header 文案：

```text
参数补全
请一次补全报价参数
按检测类型、动物实验、生物分析、报告与报价分页选择；全部补齐后统一发送给 Agent。
```

右侧显示：

```text
还需 n 项
```

### 4.4 Row

每个字段是一行 decision row：

```text
编号圆点
必填 · 计价关键字段
字段名
选项按钮组
```

Row hover：

- 轻蓝 border。
- 轻 shadow。
- 轻微 translateY。

默认折叠策略：

- 补全卡未 hover / 未 focus 时，每页只完整展示第一条字段。
- 同页后续字段只以轻微堆叠露出，不展示完整内容。
- Hover / focus 补全卡后展开当前页全部字段。
- 目的是不打断用户阅读主对话流，策略与肿瘤报告 warning 确认卡一致。

选中后：

- 写入 composer chip。
- 字段从补全卡消失。
- 删除 chip 后字段回到补全卡。

### 4.5 自定义选项

如果字段有 `自定义`：

- 不应只是普通按钮。
- 点击后按钮原位变成 input。
- 用户输入后 Enter / blur 提交。
- 提交后写入 composer chip。
- Escape 或空值 blur 可取消。

### 4.6 Footer / Pager

Footer 左侧：

```text
选择会先写入下方参数 tab，全部补齐后才正式更新右侧参数台账。
```

Footer 右侧：

```text
当前页/总页数  上一页  下一页
```

操作按钮放右下角。

报价前确认卡按钮布局：

- 左侧只放预览 icon button，不显示“预览全部参数”文字。
- 右侧放黑色主按钮 `生成报价单`。
- 预览只负责打开查看 modal；生成动作必须留在主流程卡片中。

## 5. Composer Chip

Chip 必须落在输入框内部的 tab 区，不在 composer 外部。

形态：

```text
[化合物类别：普通小分子 ×] [分析方法：LC-MS/MS ×]
```

规则：

- chip 在 composer shell 内部。
- 有 chip 时，placeholder 示例文案消失。
- 删除 chip 后，对应字段回到补全卡。
- 未发送 chip 是草稿，不更新右侧参数 panel。
- 发送后：
  - 生成用户气泡。
  - 清空 chip。
  - 更新右侧参数 panel。

发送按钮：

- 在 composer 右下角。
- 黑色方形圆角按钮。
- 使用 `Send` icon。

## 6. 右侧参数收集 Panel

### 6.1 常驻

参数 panel 必须常驻右侧。

它展示正式参数台账：

- 已提交参数。
- 已确认状态。
- 字段缺失状态。

它不展示：

- composer chip 草稿。
- 未发送选择。

### 6.2 分组

四个分组：

1. 检测类型。
2. 动物实验。
3. 生物分析。
4. 报告与报价。

每组可折叠展开。

状态：

- 已完成。
- 进行中。
- 未开始。

### 6.3 编辑入口

右侧参数行的 `Edit3` icon 不是右侧内联编辑。

点击后必须回到主对话：

1. Agent 追加一条追问：

```text
请问您希望将动物种属修改为什么？请在下方选择一个新值，发送后我会更新右侧参数。
```

2. Composer 上方出现单字段选择卡。
3. 用户选择新值。
4. 新值写入 composer chip。
5. 用户发送。
6. 右侧参数 panel 更新。

不要：

- 在右侧 panel 直接改值。
- 弹右侧小表单。
- 点击 edit 后只展开当前分组不提示用户。

## 7. Secondary Panel

### 7.1 触发

Secondary panel 默认不展示。

只能通过主对话蓝色 inline 入口触发：

- 查看过程。
- 查看规则证据。
- 查看导出校验。
- 查看产物。

Hover / focus：

- 在右侧参数 panel 下方出现。

Click：

- pin 当前 panel。

未 pin：

- 鼠标离开右侧 panel stack 后收起。

### 7.2 内容类型

处理过程：

- 展示 chain of activity。
- 类似肿瘤报告项目的校验过程。
- 有节点、竖线、简短说明。

规则证据：

- 展示字段如何影响报价。
- 展示管理费规则。
- 展示金额一致性规则。

产物与版本：

- Word 报价单。
- Excel 报价明细。
- 预览、下载入口。

### 7.3 禁止

- 不要自动打开 process panel。
- 不要用 tab 在右侧切换 primary panel。
- 不要让 secondary panel 替换参数 panel。

## 8. Activity Chain

中间对话中：

- 默认是折叠的一行。
- 文案类似：

```text
已完成报价参数更新 · 查看过程
```

点 `查看过程` 后：

- 右侧 secondary panel 展开 process chain。

生成中：

- 可以在中间短暂展开轻量 chain。
- 完成后折叠。

右侧 process chain 示例：

```text
处理过程
● 读取用户输入
  识别检测类型、动物种属、数量和周期。
● 识别 DMPK / PK 业务线
  匹配报价业务线和字段组。
● 检查计价关键字段
  确认必填字段是否齐全。
● 同步报价参数台账
  用户发送后更新右侧正式参数。
```

## 9. 报价前预览 Modal

报价前预览 modal 只做查看。

不放：

- 返回修改按钮。
- 确认生成按钮。

关闭：

- 用户点击右上角 `X`。

生成：

- 回到 composer 上方“报价前确认”卡中操作。

内容：

- 单栏即可。
- 不需要左侧分栏，因为 DMPK 参数总览一目了然。
- 表格标题：`报价参数`。
- 表格列：`类别 / 项目 / 说明`。
- 底部提示计价规则。

提示文案示例：

```text
计价关键字段已齐全。Word 报价单使用 30% 管理费，Excel 报价明细使用 15% 管理费，生成后将进行金额一致性校验。
```

## 10. 产物

生成两个产物：

- Word 报价单。
- Excel 报价明细。

生成后：

- 主对话出现生成完成提示。
- 右侧参数 panel 默认保留。
- 产物列表作为 secondary panel 通过蓝色入口 hover/pin。

Word：

- 中文 Word 不能乱码。
- 英文 Word 需要美元换算。
- 表格边框完整。

Excel：

- 金额必须和页面金额一致。
- 管理费规则明确。

金额规则：

- Excel 管理费 15%。
- Word 管理费 30%。
- 页面金额、Excel 金额、Word 金额必须一致或清楚解释差异。

## 11. 字段范围

### 11.1 PK

关键字段：

- 分子类型。
- 动物种属。
- 每组动物数。
- 组数。
- 给药周期 / 频率。
- 采样点。
- 方法类型。
- 报告格式。
- 报告语言。
- 报价区域。

### 11.2 BA Only

关键字段：

- 样品类型。
- 样本数量。
- 化合物数。
- 方法类型。
- 是否匀浆。
- 报告类型。

### 11.3 毒理

关键字段：

- 动物种属。
- 主组 / 卫星组。
- TK。
- 临床病理。
- ADA。
- 给药周期。
- 血常规点数。
- 血生化点数。
- 血凝点数。

## 12. 高风险 UX

必须防止：

- 缺字段时直接出报价。
- 未发送 chip 更新右侧参数。
- 右侧 edit 变成右侧表单编辑。
- 补全卡进聊天气泡。
- 按条数分页导致用户误解业务分类。
- 自动切页打断用户选择。
- preview modal 放确认生成按钮。
- 自动弹出右侧 process panel。
- 中文 Word 乱码。
- 空 section 出现在最终报价单。
- 猴、小鼠/大鼠、裸小鼠、转基因小鼠价格边界不清。
- 国内/欧美/亚太报价区域不明确。
- Excel 15% 和 Word 30% 管理费不明确。

## 13. 文案规则

Agent 识别后：

```text
已识别 DMPK / PK 检测、小分子、SD 大鼠、每组 2 只、2 组、试验周期 1 周和 3 个非加班采血点。还需要补充生物分析和报告与报价参数。
```

补全卡：

```text
参数补全
请一次补全报价参数
按检测类型、动物实验、生物分析、报告与报价分页选择；全部补齐后统一发送给 Agent。
```

编辑追问：

```text
请问您希望将{字段名}修改为什么？请在下方选择一个新值，发送后我会更新右侧参数。
```

报价前确认：

```text
参数已齐全，可以生成正式报价单。
请先预览完整参数和计价规则，也可以查看规则证据。确认后将生成 Word 报价单与 Excel 报价明细。
```

生成完成：

```text
报价单已生成。Word 与 Excel 金额校验一致。
```

## 14. 不做

- 不做 canvas。
- 不做估算报价。
- 不做一个问题一个问题追问。
- 不做右侧完整表单编辑。
- 不做未发送草稿同步右侧 panel。
- 不做 modal 内确认生成。
- 不做自动弹出 secondary panel。
- 不做新 sidebar 设计。

## 15. 验收清单

- 首次输入后是否识别已知字段。
- 缺失字段是否一次性进入补全卡。
- 补全卡是否在 composer 上方。
- 分页是否固定四页业务类型。
- 是否不会自动切页。
- 选项是否写入 composer chip。
- chip 是否在输入框内部。
- 有 chip 时 placeholder 是否消失。
- `自定义` 是否可输入。
- 未发送 chip 是否不会更新右侧 panel。
- 发送后是否生成用户气泡。
- 右侧参数 panel 是否常驻。
- 右侧 edit 是否回到主对话追问。
- 单字段修改卡是否出现在 composer 上方。
- 蓝色入口是否 hover/pin secondary panel。
- secondary panel 是否默认不展示。
- 报价前预览 modal 是否无确认按钮。
- 生成动作是否在主对话/composer task card。
- Word / Excel 产物是否分开展示。
- 金额和管理费规则是否明确。
